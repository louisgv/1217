Above and Beyond

  • We used localStorage to save the image hash set as well as the number of maggots on the scene.
  • We created a system that allows player to drag and drop images from the desktop (Binary form) or from other website (URL form) onto the screen.
  • We have created a large network of classes and containers. (8 classes)
  • The maggots move using a customized steering behavior that is wandering focused, which make them more interesting.
  • We integrated the web's File API with PixiJS's Loader, Texture and Sprite API
  • The maggot consumes the image to grow and multiply organically.
  • The application has sounds for the maggots moving and for the maggots eating.
  • Some of the challenges we faced was figuring out the layout of our code, getting the maggots to eat the images and learning PixiJS as we worked.
  • The work was done by Lab and Stoney Reed. The project we have created is a Tamagotchi game in which you feed your pet maggots and watch them multiply.

Notes

  • Dropping image from remote location might result in the image not showing. This is due to the CORS config of the remote host.
  • Upon resetting the game, the image that was once used can now be re-used. We don't save the image hash table into the local storage.
