Notes

  • There is no 3rd-party JavaScript library used.
  • There is a cheating mechanism for the player. Simply highlight the score cards from the NPC side to see what element does the NPC have.
  • The NPC has a fake "thinking phase" untill it makes a move. During this time, the cursor will turn into its progressing state.
  • The gifs are cached into a GIPHY_TABLE variable in localstorage, and has an expire cycle of 1 hour. This limits the amount of call necessary to Giphy while also provide optimal reloading experience.
  • The CSS uses the pointer-events attribute which is not supported by the W3 validator, but it works on most modern browser according to caniuse
  • The game uses the async/await syntax, thus will only works on browser in this chart
  • The game uses the Promise API, thus will only works on browser in this chart
  • The game uses the fetch API, thus will only works on browser in this chart
  • The game uses localstorage browser API to cache the state of the tutorial, the game as well as a giphy database.
  • The game uses Array.from ES6 feature, thus will only works on browser in this chart.
  • The logo for the favicon is my design. I created it using Piskel
  • The hero cards are my drawings.
  • The card gif images were fetch from Giphy.com, using keywords associated with each element.
  • The works done in this project is authored by lab at mail dot rit dot edu, both CSS and JavaScript.
  • License notice was taken from my blog, which is also my design.
